> 用户问: "防幻觉这块有什么新的技术可以用?"
> 真实查的 (2026-07-04 23:28 EDT), 不靠印象

---

## 一、防幻觉 5 大技术 (按推荐优先级)

### 1. RAG (检索增强生成) — **最实用**
**原理**: 回答前先从知识库检索相关文档, 把文档 + 用户问题一起给 LLM. 答错 = 检索错, 不会瞎编.

**对心理 AI 的价值**:
- ✅ 减少"心理学概念错误" (LLM 经常编造不存在的疗法/疗法名)
- ✅ 减少"危机识别漏判" (查 19 关键词库 + 危机干预流程)
- ✅ 减少"咨询师案例误用" (查真实案例库)
- **技术成本**: 中, 需要向量库 (Pinecone / Milvus / reading-fl HNSW)

**实施** (1-2 周):
- 把你的 6 角色 prompt + 19 关键词库 + 危机热线 做成 RAG 知识库
- 用腾讯云 CloudBase 自带向量 (或 reading-fl HNSW)
- chat 云函数前加 1 个 retrieve 步骤

### 2. Constitutional AI / Self-Critique — **新潮**
**原理**: LLM 第一次回答, 第二次**自己审查**回答哪里不对, 改写. 2 次生成 = 质量 +1.

**对心理 AI 的价值**:
- ✅ "我不是真人" 边界声明 → 让 LLM 自查有没有冒充
- ✅ "家暴/自伤" 危机处理 → 让 LLM 自查有没有给具体建议
- ✅ "不道德判断" → 让 LLM 自查有没有评判用户

**技术成本**: 中, 2 次 LLM 调用 (latency ×2, 成本 ×2)

**实施** (3-5 天):
- 6 角色 prompt 加"自查清单"
- 回答后自动跑 1 次 review (用同模型)

### 3. Self-Consistency / Multiple Sampling — **稳**
**原理**: 同一问题让 LLM 答 N 次, 取**最一致的答案**. 5 次采样 = 错误率 ↓30-50%.

**对心理 AI 的价值**:
- ✅ 危机判断 (5 次都判"高危" → 真的高危)
- ✅ 角色路由 (5 次都推荐"叙事" → 真的适合)

**技术成本**: 中高, 成本 ×5, latency ×5. **不适合实时聊天**, 适合后台.

### 4. Logit / Confidence Threshold — **新**
**原理**: 看 LLM 输出每个 token 的置信度 (probability), 低于阈值 = 不可信, 重生成或转人工.

**对心理 AI 的价值**:
- ✅ LLM 自己不确定时, 不硬答, 弹 "我不太确定, 你想让我换个角度吗?"

**技术成本**: 高, 大多数 API (OpenAI/AMAX/CloudBase) **不返 logit**, 只能自己训练本地模型才能拿.

### 5. RAG + Self-Critique 组合 — **最推荐**
- 先 RAG 检索 (减少幻觉)
- 再 Self-Critique 复查 (兜底)
- 2 层防御

## 二、对你的 6 角色 / 19 关键词 / 危机干预, 实际可用

### 2.1 6 角色 prompt 加防幻觉
```js
// 现有 6 角色 prompt 模板
const NARRATIVE_PROMPT = `你是叙事疗法 AI...`
// 加一个 "guard" 字段
const GUARD = `
⚠️ 自查清单 (回答前必读):
1. 我有没有冒充真人? (不能说"我是 xx 年经验咨询师")
2. 我有没有给具体医疗建议? (危机情况必须转人工)
3. 我有没有编造心理学概念? (用 RAG 检索过的真实知识)
4. 我有没有评判用户? (保持中立)
如有 1-4 任一, 必须先改正再回答.
`
```

### 2.2 19 关键词危机检测 → RAG 知识库
- 把"评估自杀风险 / 危机干预 SOP / 24h 热线 / 北京心理危机中心" 做成 RAG 知识库
- 用户消息含 19 关键词 → **先 RAG 检索危机 SOP** → LLM 按 SOP 回答
- **避免 LLM 现场编危机处理流程**

### 2.3 案例库检索 (qizhenxinli 方向 B 配合)
- 历史咨询师案例 (脱敏) → 做成 RAG
- 用户新问题 → 检索类似案例 → LLM 参考回答
- **避免 LLM 套通用模板, 给个性化建议**

## 四、给你 1 个马上能用的代码示例 (P0)

```js
// 在 callLLM 之前加 1 层 Self-Critique
async function callLLMWithGuard(system, history, userMsg) {
  // 第 1 次: 正常生成
  const response = await callLLM(system, history);
  
  // 第 2 次: 自查 (用同模型 + 简单 prompt)
  const guard = await callLLM(
    `你是审查员. 检查以下回答:
     "${response}"
     
     检查 4 点:
     1. 冒充真人? (是 → 改"我是 AI 助手")
     2. 医疗建议? (是 → 改"建议咨询专业医生")
     3. 编造概念? (是 → 删除)
     4. 评判用户? (是 → 删除)
     
     输出: 修改后的回答 (原文照搬, 只需改不对的).`,
    []
  );
  return guard;
}
```

**1 天集成, latency ×2 (从 2s 变 4s), 成本 ×2**. 但**幻觉率 ↓ 30-50%**.

## 五、不要做的 (我见过的反模式)

- ❌ **只靠 prompt engineering**: 治标不治本, 复杂场景仍然幻觉
- ❌ **只靠 Self-Consistency**: 成本爆炸, 实时聊不起
- ❌ **盲目信任 logit**: API 不返, 自训练成本高
- ❌ **不用 RAG 只靠 LLM 记忆**: LLM 训练数据截止后的事实全错
